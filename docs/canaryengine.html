Introduction

CanaryEngine is a game framework developed by Canary. The setup is quite easy actually, first you must download the plugin: Canary Studio. For the next steps, please view the video below:

Here are a few advantages to using CanaryEngine over other frameworks like Knit and others.

  • Fully typed
  • Simple API
  • Fully Documented
  • Easy Setup
  • Nicely Organized
  • Super Lightweight
  • Performant
Being performant is a large one for a framework. Though, in CanaryEngine, all setup is done on the server, and is only done a single time. This is one of the many reasons that makes CanaryEngine more performant than some alternative frameworks out there. The networking system is also something we kept in mind while creating the framework. We always thought that Roblox's RemoteEvents were annoying to work with due to the fact that you have to reference them and create them multiple times. They are also extremely slow when for use cases like head rotation. CanaryEngine's networking system is crazy fast and crazy simple, avoiding the use of tuples for a better type checking experience.

API Reference

Package.GetEngineServer()

Description:

Allows you to use the server sided version of cEngine.

Params:

  • None

Returns:

  • EngineServer: dictionary
Package.GetEngineClient()

Description:

Allows you to use the client sided version of cEngine.

Params:

  • None

Returns:

  • EngineClient: dictionary
Package.CreateSignal()

Description:

Creates a ScriptSignal, can be used in the same RunContext.

Params:

  • None

Returns:

  • ScriptSignal: CustomScriptSignal
Package.GetLatestPackageVersionAsync()

Description:

Returns the latest version of a package. You can also log a warning in the console if you wish. If the web request fails, the function will return nil.

Params:

  • package: Instance
    The package thats version should be checked.
  • warnIfNotLatestVersion: boolean?
    Tells whether if a warning should be displayed in the output if the package isn't up to date.

Returns:

  • PackageVersion: number?
EngineServer.CreateNetworkSignal()

Description:

Creates a NetworkSignal, can be used in the other types of RunContexts. Creating another NetworkSignal in another script with name name will allow you to fire and connect to it.

Params:

  • name: string
    The name of the new signal.

Returns:

  • NetworkSignal: ServerNetworkSignal
EngineClient.CreateNetworkSignal()

Description:

Creates a NetworkSignal, can be used in the other types of RunContexts. Creating another NetworkSignal in another script with name name will allow you to fire and connect to it.

Params:

  • name: string
    The name of the new signal.

Returns:

  • NetworkSignal: ClientNetworkSignal

Libraries

Package.Utility

Description:

The Utility library, comes with utility functions such as xassert() and ArrayToDictionary()

Package.Benchmark

Description:

The Benchmarking library, useful for benchmarking (hence the name), comes with a constructor and things like BenchmarkObject:Start() and BenchmarkObject:Start().

Package.Serialize

Description:

The Serialize library is useful for serializing primitive types that need to be stored in a data store or sent across server-client boundaries. Uses BlueSerializer by @commitblue.
